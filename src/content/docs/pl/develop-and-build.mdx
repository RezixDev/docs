Masz już projekt Astro? To czas zacząć budować! 🚀

## Edytuj swój projekt

Aby wprowadzić zmiany w swoim projekcie, otwórz folder projektu w edytorze kodu. Praca w trybie develop z uruchomionym serwerem dev pozwala widzieć aktualizacje strony podczas edycji kodu.

Możesz również [dostosować swoje środowisko deweloperskie](#configure-your-dev-environment), np. skonfigurować TypeScript lub zainstalować oficjalne rozszerzenia edytora dla Astro.

### Uruchom serwer dev Astro

Astro posiada wbudowany serwer deweloperski, który ma wszystko, czego potrzebujesz do pracy nad projektem. Polecenie CLI `astro dev` uruchamia lokalny serwer deweloperski, dzięki czemu będziesz mógł po raz pierwszy zobaczyć swoją nową stronę w akcji.

Każdy szablon startowy zawiera wstępnie skonfigurowany skrypt, który uruchomi `astro dev` za Ciebie. Po przejściu do katalogu projektu, użyj swojego ulubionego menedżera pakietów, aby uruchomić to polecenie i wystartować serwer deweloperski Astro.

<Tabs>
  <TabItem label="npm">
```shell
npm run dev
```
  </TabItem>
  <TabItem label="pnpm">
```shell
pnpm run dev
```
  </TabItem>
  <TabItem label="yarn">
```shell
yarn run dev
```
  </TabItem>
</Tabs>

Jeśli wszystko przebiegło pomyślnie, Astro będzie teraz wyświetlać Twój projekt pod adresem [http://localhost:4321/](http://localhost:4321/). Odwiedź ten link w przeglądarce i zobacz swoją nową stronę!

### Pracuj w trybie deweloperskim

Astro będzie nasłuchiwać na żywo zmian plików w katalogu src/ i aktualizować podgląd strony podczas budowania, więc nie będziesz musiał restartować serwera podczas wprowadzania zmian w trakcie rozwoju. Zawsze będziesz mógł zobaczyć aktualną wersję swojej strony w przeglądarce, gdy serwer deweloperski jest uruchomiony.

Podczas przeglądania strony w przeglądarce będziesz mieć dostęp do [paska narzędzi deweloperskich Astro](/en/guides/dev-toolbar/). Podczas budowania pomoże Ci to sprawdzić Twoje [wyspy](/en/concepts/islands/), wykryć problemy z dostępnością i więcej.

Jeśli nie możesz otworzyć swojego projektu w przeglądarce po uruchomieniu serwera deweloperskiego, wróć do terminala, w którym uruchomiłeś polecenie dev i sprawdź wyświetlony komunikat. Powinien powiedzieć Ci, czy wystąpił błąd, czy Twój projekt jest serwowany pod innym adresem URL niż [http://localhost:4321/](http://localhost:4321/).

## Zbuduj i zobacz podgląd swojej strony

Aby sprawdzić wersję swojej strony, która zostanie utworzona w czasie budowania, zakończ działanie serwera deweloperskiego (<kbd>Ctrl</kbd> + <kbd>C</kbd>) i uruchom odpowiednie polecenie budowania dla swojego menedżera pakietów w terminalu:

<Tabs>
  <TabItem label="npm">
  
```shell
npm run build
```

  </TabItem>
  <TabItem label="pnpm">
  
```shell
pnpm build
```

  </TabItem>
  <TabItem label="yarn">
  
```shell
yarn run build
```

  </TabItem>
</Tabs>

Astro zbuduje gotową do wdrożenia wersję Twojej strony w osobnym folderze (domyślnie dist/) i możesz obserwować jej postęp w terminalu. To ostrzeże Cię o wszelkich błędach budowania w Twoim projekcie, zanim wdrożysz na produkcję. Jeśli TypeScript jest skonfigurowany na strict lub strictest, skrypt budowania sprawdzi także Twój projekt pod kątem błędów typów.

Gdy budowanie zostanie zakończone, uruchom odpowiednie polecenie podglądu (np. npm run preview) w terminalu i możesz zobaczyć zbudowaną wersję swojej strony lokalnie w tym samym oknie podglądu przeglądarki.

Zauważ, że to pokazuje Twój kod taki, jaki istniał, gdy polecenie budowania zostało ostatnio uruchomione. Ma to na celu danie Ci podglądu tego, jak Twoja strona będzie wyglądać, gdy zostanie wdrożona w sieci. Wszelkie późniejsze zmiany, które wprowadzisz do swojego kodu po budowaniu, **nie** będą odzwierciedlone podczas podglądu strony, dopóki nie uruchomisz ponownie polecenia budowania.

Użyj (<kbd>Ctrl</kbd> + <kbd>C</kbd>), aby zakończyć podgląd i uruchomić inne polecenie terminala, takie jak ponowne uruchomienie serwera deweloperskiego, aby wrócić do [pracy w trybie deweloperskim](#pracuj-w-trybie-deweloperskim), który aktualizuje się podczas edycji, aby pokazać podgląd na żywo zmian w kodzie.

<ReadMore>Przeczytaj więcej o [CLI Astro](/en/reference/cli-reference/) i poleceniach terminala, których będziesz używać podczas budowania z Astro.</ReadMore>

:::tip 
Możesz chcieć [wdrożyć swoją nową stronę od razu](/en/guides/deploy/), zanim zaczniesz dodawać lub zmieniać zbyt dużo kodu. Jest to pomocne, aby opublikować minimalną, działającą wersję swojej strony i może zaoszczędzić Ci dodatkowy czas i wysiłek na rozwiązywanie problemów z wdrożeniem później.
:::

Zbuduj w pełni funkcjonalny blog Astro, zaczynając od pojedynczej pustej strony w naszym [samouczku wprowadzającym](/en/tutorial/0-introduction/).
